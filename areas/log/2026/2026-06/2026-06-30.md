## Focus
- Fix projector-queue / to-projector conversion issues for MAGCubic projector

## Log

### Session 19:16 (canticos)
- Debugged `projector-queue` and `to-projector` in `canticos/bin/` after user reported video conversion produces corrupt/broken output on MAGCubic projector
- **Root cause 1 — MMCO corruption**: ORGANiC scene releases have broken H.264 streams (mmco: unref short failure, non-monotonic DTS). Stream-copy preserved these, causing 0.5s duration loss. Fix: added `ffprobe` MMCO detection during probe — when found, falls back to VAAPI re-encode instead of stream copy.
- **Root cause 2 — Level 4.2**: VAAPI encoder auto-selected H.264 Level 4.2, but MAGCubic only supports up to 4.0/4.1. Fix: added `-level 40` to all 3 VAAPI encode paths.
- **Root cause 3 — A/V sync on MP4**: The projector's MP4 player doesn't handle H.264 decoder delay PTS offsets (~1.5s) correctly, but the MKV player does. Fix: changed output container from MP4 to MKV.
- Final fix (commits `30b2d00`, `d5c123d`): MMCO detection + Level 40 + MKV output
- Verified MKV output plays correctly locally; user confirmed MAGCubic playback works

## Decisions
- Use MKV instead of MP4 for projector output — the MAGCubic's MKV player handles PTS offsets correctly
- MMCO-corrupted H.264 must be re-encoded (not stream-copied) — detection via ffprobe stderr
- VAAPI hardware encoding must be constrained to Level 4.0 for MAGCubic compatibility
